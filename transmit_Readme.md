# VirtualTabs — Transmit Files 功能說明

## 功能緣起

在 VS Code 中開發完成的程式碼，最終往往需要部署到其他電腦，或安裝到特定軟體的子目錄中（例如 CNC 控制器、HMI 設備、嵌入式系統等）。

**以往的繁瑣流程：**
1. 在 VS Code 完成編輯
2. 另開啟檔案總管，手動導覽到專案目錄
3. 選取並複製目標檔案
4. 切換到目標電腦／軟體目錄
5. 貼上並覆寫舊版本
6. 重複上述步驟……

**追加此功能的目的：**
將上述步驟簡化為「右鍵 → 傳送檔案」一個動作，預先設定好專案的目標部署目錄，日後每次更新只需按一下右鍵即可完成。

---

## 功能介紹

### 傳送單一檔案 / 多選檔案

在 VirtualTabs 側邊欄中，對任意檔案右鍵，選擇 **Transmit Files（傳送檔案）**：

- 若已在側邊欄多選檔案（Ctrl + 點擊），則一次傳送所有選取的檔案
- 若未多選，則僅傳送右鍵點擊的單一檔案

### 傳送整個群組

對 VirtualTabs 中的**群組**（包含自訂群組和內建群組）右鍵，選擇 **Transmit Files（傳送檔案）**，將群組內**所有檔案**一次傳送至目標目錄。

### 目標選擇

- 設定檔中只有一個目標時，自動選取，不顯示選單
- 設定檔中有多個目標時，彈出 **QuickPick 選單**，讓使用者選擇本次要部署到哪個目錄

### 傳送進度

批次傳送時顯示進度通知，包含：
- 目前傳送的檔案名稱
- 進度計數（第 N / 共 M 個）
- 支援**取消**操作，中斷後顯示已成功傳送的數量

### 覆寫確認

目標目錄中若已存在同名檔案，會彈出確認對話框：
- **覆寫（Overwrite）**：以新版本取代
- **跳過（Skip）**：保留目標目錄中的現有檔案

### 目標目錄自動建立

若設定的目標路徑尚不存在，傳送時會自動建立該目錄（含多層子目錄）。

---

## 設定方式

在專案的 `.vscode/` 目錄下建立 `transmitConfig.json`：

```json
{
    "transmitTargets": [
        {
            "name": "生產機台 A",
            "path": "D:\\MachineSoftware\\programs"
        },
        {
            "name": "備份伺服器",
            "path": "\\\\server01\\share\\backup"
        }
    ]
}
```

| 欄位 | 說明 |
|------|------|
| `name` | 顯示名稱，出現在 QuickPick 選單中 |
| `path` | 目標目錄的絕對路徑（本機或網路路徑均可） |

> **備用設定**：若 `transmitConfig.json` 不存在，也可將 `transmitTargets` 直接加入現有的 `.vscode/virtualTab.json` 中。

## 操作流程示意

```
VS Code 側邊欄 (VirtualTabs)
│
├── 📁 MyGroup
│   ├── 📄 main.c          ← 右鍵 → Transmit Files
│   ├── 📄 config.h
│   └── 📄 utils.c
│
└── 右鍵群組 → Transmit Files  （傳送群組內全部檔案）

          ↓ 選擇目標（多目標時顯示 QuickPick）

    ┌─────────────────────────┐
    │  生產機台 A              │
    │  D:\MachineSoftware\... │
    │                         │
    │  備份伺服器              │
    │  \\server01\share\...   │
    └─────────────────────────┘

          ↓ 傳送中（顯示進度通知）

    ✓ Transmitted 3 of 3 files to '生產機台 A'
```

## 注意事項

- 傳送功能為**複製**操作，不會刪除來源檔案
- 僅傳送檔案本身，不會重建目錄結構（所有檔案平鋪到目標目錄）
- 目標路徑支援 UNC 網路路徑（`\\server\share\...`）
- 設定檔建議加入 `.gitignore`，避免將本機路徑提交到版本控制
